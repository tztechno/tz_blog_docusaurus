Push Docusurus

June 22, 2024

stpete

ローカルで作成したDocusaurusプロジェクトをGitHub Pagesに公開する方法を説明しますね。

手順概要

  1. Docusaurusプロジェクトの作成 ローカルでDocusaurusプロジェクトを作成し、必要なページやコンテンツを追加・編集します。

  2. GitHubリポジトリの作成 GitHub上に新しいリポジトリを作成します。このリポジトリでDocusaurusプロジェクトをホストします。

  3. GitHubリポジトリとローカルリポジトリの紐付け ローカルのDocusaurusプロジェクトをGitHubリポジトリに関連付けます。

  4. プロジェクトのビルド Docusaurusプロジェクトをビルドし、生成された静的ファイルをGitHub Pagesでホストできる形式にします。

  5. GitHub Pagesの設定 GitHubリポジトリの設定で、GitHub Pagesを有効化し、どのブランチ（通常はgh-pagesブランチかdocsフォルダ）を公開するか設定します。

  6. プッシュ ローカルでビルドしたDocusaurusの静的ファイルをGitHubリポジトリにプッシュします。

より詳細な手順

1. Docusaurusプロジェクトの作成

まず、Docusaurusのプロジェクトを作成します。以下のコマンドを実行します：

npx @docusaurus/init@latest init my-docusaurus-project classic
cd my-docusaurus-project

2. GitHubリポジトリの作成

GitHubで新しいリポジトリを作成します。リポジトリ名を決め、必要に応じてREADMEや.gitignoreを設定します。

3. GitHubリポジトリとの紐付け

ローカルのDocusaurusプロジェクトをGitHubリポジトリに関連付けます。以下のようにコマンドを実行します：

git remote add origin <GitHubリポジトリのURL>
git branch -M main  # ブランチ名をmainに変更（もし必要ならば）
git push -u origin main

4. プロジェクトのビルド

Docusaurusプロジェクトをビルドします。以下のコマンドを実行します：

npm run build

これにより、/buildディレクトリにビルドされた静的ファイルが生成されます。

5. GitHub Pagesの設定

GitHubリポジトリの設定ページに移動し、以下の設定を行います：

  • Settings > Pagesに移動
  • Sourceを設定し、GitHub Pagesを有効化します。通常はgh-pagesブランチまたはdocsフォルダを指定します。
  • Saveをクリックして設定を保存します。

6. プッシュ

最後に、ビルドされた静的ファイルをGitHubリポジトリにプッシュします：

git add .
git commit -m "Build Docusaurus for GitHub Pages"
git push origin main

これで、GitHub PagesでDocusaurusプロジェクトがホストされるようになります。少し時間がかかる場合がありますが、設定が反映されたら、GitHub PagesのURLでサイトにアクセスできます。

以上が、DocusaurusプロジェクトをローカルからGitHub Pagesに公開する手順です。
